# 測試規格文件（TEST_SPEC）

**專案**：巴哈姆特電玩資訊站 E2E 自動化測試
**版本**：v1.0.0
**日期**：2026-05-14
**負責人**：企劃 / QA 團隊
**測試範圍**：SCOPE REDUCTION MVP — 4 個核心頁面元素存在性驗證

---

## 文件目的

本規格文件定義每個測試頁面的：
- 測試目標與範圍
- 待驗證的具體 UI 元素（含 XPath 定位策略說明）
- 預期結果
- 邊界條件與特殊情境

開發人員依據本文件實作 Page Object 與測試腳本，QA 人員依據本文件驗收測試結果。

---

## 全域測試前置條件

| 項目 | 說明 |
|------|------|
| 瀏覽器 | Google Chrome（Headless 或可視化模式） |
| 視窗尺寸 | 1920 x 1080（模擬桌面使用者） |
| 等待策略 | 使用 Explicit Wait，最長等待 15 秒 |
| 失敗處理 | 元素未找到或逾時，測試標記為 FAIL 並截圖 |
| 網路條件 | 需要正常連線，不模擬弱網環境 |

## TC-01：首頁核心元素驗證

**測試目標**
驗證巴哈姆特首頁（https://www.gamer.com.tw/）在正常載入後，主要導覽列與網站 Logo 元素確實存在於 DOM 中。

**測試 URL**
```
https://www.gamer.com.tw/
```

### 待驗證元素

#### 元素 1：網站 Logo

| 項目 | 說明 |
|------|------|
| 元素描述 | 頁面頂部的巴哈姆特 Logo 圖片或連結 |
| 定位策略 | 優先使用 `id` 屬性定位（若 Logo 元素有 `id="logo"` 或類似屬性）；其次使用帶語義的 `class` 屬性（如 `class` 包含 `logo`）；以 `<img>` 的 `alt` 屬性包含「巴哈姆特」作為備用 |
| XPath 說明 | 尋找 `<header>` 或頂部 `<div>` 內，`href` 指向首頁根路徑（`/` 或 `https://www.gamer.com.tw/`）的 `<a>` 標籤，其子節點包含 `<img>` 元素 |
| 不使用 | 絕對位置 XPath（如 `/html/body/div[1]/div[2]/a`），因版面調整後易碎 |

#### 元素 2：主導覽列（Navigation Bar）

| 項目 | 說明 |
|------|------|
| 元素描述 | 頁面頂部橫向導覽選單，包含多個主要分類連結（如「首頁」、「新聞」、「討論板」等） |
| 定位策略 | 優先使用語義化 HTML 標籤 `<nav>`；若無 `<nav>` 則尋找包含 `nav` 或 `menu` 字樣的 `id` 或 `class` 屬性 |
| XPath 說明 | 尋找 `<nav>` 元素本身存在，或尋找 `id` / `class` 包含 `nav` 關鍵字的 `<ul>` 或 `<div>` 容器，確認其子節點中包含多個 `<li>` 或 `<a>` 連結項目 |
| 不使用 | 依賴特定文字內容比對（如 `text()='首頁'`），因文字可能隨版面調整而變動 |

### 預期結果

| 驗證項目 | 預期結果 |
|----------|----------|
| 頁面載入狀態 | HTTP 200，頁面標題（`<title>`）包含「巴哈姆特」 |
| Logo 元素 | 元素存在於 DOM，`isDisplayed()` 回傳 `true` |
| 導覽列元素 | 元素存在於 DOM，`isDisplayed()` 回傳 `true`，且子連結數量大於等於 3 |

### 邊界條件

- 若遭遇 Cloudflare Challenge 頁面，`<title>` 不含「巴哈姆特」，測試應標記為 SKIP 並記錄警告，非 FAIL。
- 若網站正在維護，伺服器回傳 503，測試標記為 FAIL 並附上 HTTP 狀態碼說明。
- Logo 圖片本身是否正確顯示（非破圖）不在本次 MVP 驗證範圍內。

## TC-02：討論板帖子列表驗證

**測試目標**
驗證巴哈姆特討論板（https://forum.gamer.com.tw/）在正常載入後，帖子列表容器存在，且至少包含一筆帖子項目。

**測試 URL**
```
https://forum.gamer.com.tw/
```

### 待驗證元素

#### 元素 1：帖子列表容器

| 項目 | 說明 |
|------|------|
| 元素描述 | 頁面中承載所有帖子列表的外層容器元素 |
| 定位策略 | 尋找 `id` 包含 `forum`、`list`、`post` 等關鍵字的 `<div>` 或 `<ul>` 元素；或具有 `class` 包含 `forum-list`、`topic-list` 的容器 |
| XPath 說明 | 定位方式為：在頁面 `<main>` 或主要內容區塊中，尋找包含多個列表子項目（`<li>` 或特定 `<div>` 行）的外層容器，確認其存在即可 |
| 不使用 | 依賴第幾個 `<div>` 的絕對路徑定位 |

#### 元素 2：帖子列表項目（至少一筆）

| 項目 | 說明 |
|------|------|
| 元素描述 | 列表中的單一帖子項目，通常包含帖子標題連結、發文時間、回覆數等資訊 |
| 定位策略 | 在帖子列表容器內，尋找重複出現的 `<li>` 或特定 `class` 的 `<div>` 子元素；以帖子標題 `<a>` 連結的存在作為確認依據 |
| XPath 說明 | 在列表容器的直接子節點中，尋找至少一個包含 `<a>` 標籤（href 指向討論帖詳細頁，通常含 `/C.php` 或 `/B.php` 路徑）的項目 |
| 不使用 | 直接比對帖子標題文字內容（文字每日不同） |

### 預期結果

| 驗證項目 | 預期結果 |
|----------|----------|
| 頁面載入狀態 | HTTP 200，頁面標題包含「討論板」或「巴哈姆特」 |
| 帖子列表容器 | 元素存在且可見 |
| 帖子列表項目 | 至少找到 1 個帖子項目元素，數量大於等於 1 |

### 邊界條件

- 若討論板因時段清空導致帖子數為 0，測試應標記為 WARN 而非 FAIL（此情況極罕見）。
- 帖子內容（標題文字、時間等）正確性不在本次驗證範圍，僅驗證結構存在。
- 分頁機制是否正常運作不在 MVP 範圍內。

## TC-03：動畫瘋影片列表驗證

**測試目標**
驗證動畫瘋（https://ani.gamer.com.tw/）頁面正常載入後，影片縮圖列表或影片卡片容器存在，且至少包含一個影片項目。

**測試 URL**
```
https://ani.gamer.com.tw/
```

### 待驗證元素

#### 元素 1：影片列表容器

| 項目 | 說明 |
|------|------|
| 元素描述 | 承載影片卡片的外層區塊，通常為格狀（Grid）或列狀（List）排版 |
| 定位策略 | 尋找 `id` 或 `class` 包含 `anime-list`、`video-list`、`catalog`、`content` 等關鍵字的容器元素 |
| XPath 說明 | 在頁面主要內容區（`<main>` 或 `id="main"` 的 `<div>`）中，尋找包含多個影片卡片子節點的 `<ul>` 或 `<div>` 容器 |
| 不使用 | 以 `<script>` 或動態注入資料的 JSON 內容作為驗證依據 |

#### 元素 2：影片縮圖 / 影片項目（至少一個）

| 項目 | 說明 |
|------|------|
| 元素描述 | 單一影片卡片，通常包含縮圖圖片（`<img>`）與影片標題連結（`<a>`） |
| 定位策略 | 在影片列表容器內，尋找包含 `<img>` 子節點的 `<li>` 或 `<div>` 卡片元素；縮圖 `<img>` 的 `src` 屬性應包含圖片路徑（非空值） |
| XPath 說明 | 定位列表容器的第一個子項目，確認其內部存在 `<img>` 標籤，且 `<a>` 的 `href` 指向動畫瘋的播放頁（通常含 `/animeVideo.php` 或類似路徑） |
| 不使用 | 比對縮圖圖片的具體 URL 或影片名稱文字 |

### 預期結果

| 驗證項目 | 預期結果 |
|----------|----------|
| 頁面載入狀態 | HTTP 200，頁面標題包含「動畫瘋」 |
| 影片列表容器 | 元素存在且可見 |
| 影片縮圖項目 | 至少找到 1 個影片卡片，其中包含 `<img>` 元素且 `src` 屬性非空 |

### 邊界條件

- 動畫瘋部分內容需要登入才能觀看，但首頁列表本身應為公開可見，若首頁重新導向至登入頁，測試標記為 FAIL 並記錄 URL 變化。
- 影片縮圖圖片是否正常顯示（非破圖）不在 MVP 驗證範圍，僅驗證 `<img>` 元素存在且 `src` 非空。
- JavaScript 非同步載入（懶加載）情境：需使用 Explicit Wait 等待列表容器出現，再擷取子項目，避免在 JS 尚未執行完成時就進行斷言。

## TC-04：哈啦板版塊列表驗證

**測試目標**
驗證哈啦板（https://forum.gamer.com.tw/B.php）頁面正常載入後，版塊列表容器存在，且至少包含一個版塊項目。

**測試 URL**
```
https://forum.gamer.com.tw/B.php
```

### 待驗證元素

#### 元素 1：版塊列表容器

| 項目 | 說明 |
|------|------|
| 元素描述 | 承載所有遊戲版塊（Board）項目的外層容器，通常為格狀或多欄排版 |
| 定位策略 | 尋找 `id` 包含 `board`、`list`、`baha` 等關鍵字的容器；或 `class` 包含 `board-list`、`game-list` 的元素 |
| XPath 說明 | 在頁面主要內容區中，尋找子節點為多個版塊卡片（重複結構的 `<li>` 或 `<div>`）的外層元素，每個子節點內應包含版塊名稱的 `<a>` 連結 |
| 不使用 | 以絕對路徑或 DOM 層級索引定位 |

#### 元素 2：版塊項目（至少一個）

| 項目 | 說明 |
|------|------|
| 元素描述 | 單一版塊（Board）卡片，通常包含版塊圖示或遊戲封面、版塊名稱連結 |
| 定位策略 | 在版塊列表容器內，尋找 `<a>` 元素，其 `href` 指向特定版塊頁面（通常含 `?bsn=` 查詢參數）；以包含 `bsn` 參數的連結存在作為版塊項目確認依據 |
| XPath 說明 | 容器內的第一個版塊連結，其 `href` 屬性值包含字串 `bsn=`，代表指向特定版塊的有效連結 |
| 不使用 | 比對版塊名稱文字（版塊名稱因遊戲上下架而變動） |

### 預期結果

| 驗證項目 | 預期結果 |
|----------|----------|
| 頁面載入狀態 | HTTP 200，頁面標題包含「哈啦板」或「巴哈姆特」 |
| 版塊列表容器 | 元素存在且可見 |
| 版塊項目 | 至少找到 1 個 `href` 含 `bsn=` 參數的版塊連結，數量大於等於 1 |

### 邊界條件

- 哈啦板頁面（`B.php`）若未帶任何 `bsn` 參數，預期顯示所有版塊總覽，若跳轉至其他頁面，測試標記為 FAIL 並記錄實際 URL。
- 版塊排序、圖示、數量等細節不在 MVP 驗證範圍。
- 若頁面需要登入才能查看版塊列表，測試標記為 FAIL 並說明需要認證的情況。

## 測試執行順序與相依性

| 執行順序 | 測試案例 | 相依性 |
|----------|----------|--------|
| 1 | TC-01 首頁 | 無（基礎驗證） |
| 2 | TC-02 討論板 | 無（獨立頁面） |
| 3 | TC-04 哈啦板 | 建議在 TC-02 後執行（同域名，確認 forum 子域正常後再測） |
| 4 | TC-03 動畫瘋 | 無（獨立子域） |

> 各測試案例之間無功能相依性，可平行執行。執行順序建議僅為除錯方便考量。

## 測試通過標準（Definition of Done）

所有測試案例須同時滿足以下條件，方視為本次 MVP 測試通過：

1. TC-01 至 TC-04 全數標記為 **PASS**（無 FAIL 項目）。
2. 每個 PASS 案例附有對應的 **瀏覽器畫面截圖說明**（文字描述截圖內容）。
3. 每個案例說明 **實際行為 vs 預期行為** 是否一致，若一致需說明具體原因。
4. 若有 WARN 或 SKIP 項目，需附上說明且不影響整體結論。
